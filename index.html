Subsecuencia común más larga

Un algoritmo común dentro de las ciencias de la computación

¿Qué es el algoritmo de la subsecuencia común más larga?

El algoritmo de la SC+L (LCS para los amigos) es un algoritmo que busca una secuencia de carácteres dentro de un conjunto de "palabras". Esta secuencia de caracteres que busca, debe de ser la más larga posible.

Por ejemplo, para las palabras "Aguacate" y "Cacahuate", la secuencia de carácteres más larga es "auate". Como ves, no deben de ser necesariamente letras consecutivas.

¿Cómo se obtiene ese resultado?

Bueno, hay varias formas de hacerlo. Quizá ya te hayas imaginado esa forma, por eso aquí te la presentamos. Esto que verás aquí es un código que tu computadora puede entender (JavaScript) y el cual ejecutará para resolver este problema.

Forma normal

Corto pero lento

Resultado: 

function fuerzaBrutaMSS(arreglo, tam){
    var cont;
    for (var i = 0; i < tam; i++){
        cont = 0;
        for (var j = i; j < tam; j++){
            cont = cont + parseInt(arreglo[j]);
            if (cont > aux){
                aux = cont;
            }
        }
    }
    return aux;
}

Forma optimizada

Largo pero rápido

Resultado: 

function divideMSS(arreglo, inicio, fin){
if(inicio == fin){
    return arreglo[inicio];
}
var mitad = Math.trunc((inicio+fin)/2);

var prefijo = 0,
    sufijo = 0,
    suma, izq, der, centro;
var izq = divideMSS(arreglo, inicio, mitad);
var der = divideMSS(arreglo, mitad+1, fin);

suma = 0;
sufijo = arreglo[mitad];
for (var i = mitad; i >= inicio ; i--){
    suma += arreglo[i];
    if(suma > sufijo){
        sufijo = suma;
    }
}

suma = 0;
prefijo = arreglo[mitad+1];
for (var i = mitad + 1; i <= fin; i++){
    suma += arreglo[i];
    if(suma>prefijo){
        prefijo = suma;
    }
}
centro = sufijo + prefijo;
return Math.max(izq,der,centro);
}

Entonces... ¿Cuál es la mejor forma para resolver este problema?

Con este algoritmo optimizado, el cuál esta construido utilizando la técnica de "programación dinámica".

¿Por qué este último algoritmo es más rápido que el anterior?

Bueno, esto se debe a que este algoritmo divide todo el problema (Las palabras) en problemas más chiquitos, tan chiquitos que son muy sencillos de resolver. En este caso, la palabra más chiquita es aquella que esta formada de... ¡Ninguna letra!

Quizá esto último te suene raro, pero para una computadora, una palabra formada de cero letras es totalmente válido.

Sin embargo, hay veces en las que la computadora resuelve estos "mini-problemas" 2 o más veces, lo cuál gasta tiempo inútilmente.

Para esto, existe la programación dinámica.

Cuando la computadora se encuentra con un "mini-problema" que ya resolvió antes, en vez de volverlo a resolver, obtiene a la solución que previamente guardó cuando calculó el mini-problema por primera vez.
